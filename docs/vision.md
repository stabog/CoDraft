# Видение продукта

## Проблема

Команды обмениваются документами через Word и почту: непонятно, кто что изменил, комментарии теряются, параллельные правки конфликтуют.

## Решение

Единый документ в облаке с:

- **версиями** — осмысленные снимки по явной фиксации;
- **комментариями** — к конкретной версии, с привязкой к фрагменту текста;
- **сравнением** — diff между версиями (было / стало);
- **участниками** — люди и, возможно, LLM-агенты.

## Два режима сотрудничества (не смешивать)

При создании документа выбирается **один** режим. Переключение после создания — дорого, на старте не планируется.

### Async (как Word) — текущий фокус

- Один **shared draft** — общая редактируемая копия (для тех, у кого есть право править).
- **Версии** — ручная фиксация результата («зафиксировать»).
- Комментарии к **замороженным версиям**.
- Diff между версиями, опционально merge предложений.
- Без realtime-синхронизации на первом этапе.

Подрежимы async описаны в [async-workflows.md](./async-workflows.md).

### Live (как Google Docs) — отложено

- Один живой документ, синхронизация в реальном времени (WebSocket, CRDT/OT).
- История — лента правок, не ручные «коммиты».
- Комментарии к текущему состоянию, outdated при правках.
- Merge-ветки не нужны.

Отдельный движок синхронизации и истории. Общий с async только каркас: редактор, список документов, экспорт.

## Целевые сценарии async

| Сценарий | Подрежим |
|----------|----------|
| Двое соавторов, пинг-понг правок | Handoff |
| Автор + много ревьюеров / LLM | Owner hub |
| Договор, политика, бриф | Owner hub |
| Параллельные правки без арбитра | Покрывается owner hub через proposals; чистый DAG (open) — только при необходимости |
